I

L'Estructura "Trinitat" (HTML / CSS / JS)

L'esquelet, el vestit i el cervell — tot en un sol fitxer

Una pàgina web professional sempre té tres parts: l'esquelet (HTML), el vestit (CSS) i el cervell (JS). En aquest bloc veuràs com conviuen tots tres dins d'un sol fitxer.

bloc-I-trinitat.html 
<!DOCTYPE html>
<html>
<head>
    <title>La meva primera web</title>
    <style>
        /* AQUÍ VA EL CSS (EL VESTIT) */
        h1 { color: #e67e22; text-align: center; font-family: sans-serif; }
    </style>
    <script>
    // AQUÍ VA EL JAVASCRIPT (EL CERVELL)
    console.log("Hola! La pàgina s'ha carregat correctament.");
    </script>
</head>
<body>

    <h1>Soc una pàgina HTML</h1>
   
</body>
</html>

La Configuració "Pro" (Metadades, UTF-8 i Bootstrap)

El <head> que prepara la web per al món real

Dins del <head> preparem la web per al món real: accents que es veuen bé, connexió a llibreries externes (Bootstrap) i com es veurà el link quan el compartim per WhatsApp.

Anatomia de l'Etiqueta (Atributs, Classes i ID)

Com les etiquetes obtenen "superpoders"

Les etiquetes no només diuen "què són", sinó "com es comporten". Fem servir Atributs per donar informació extra, Classes per agrupar estils i ID per identificar un element únic.

<!-- 1. DIV: El contenidor (Bloc) -->
<div id="seccio-principal" class="caixa-estil">
    
    <!-- 2. IMG: Imatge amb atributs (src, alt, width) -->
    <img src="https://picsum.photos/150" alt="Imatge d'exemple" width="100">

    <!-- 3. BR: Salt de línia (sense tancament) -->
    <br>

    <!-- 4. A: Enllaç amb atribut ID per a JS -->
    <a href="https://agora.xtec.cat" id="enllac-escola">Visita la nostra web</a>

    <!-- 5. BUTTON: Interacció amb classe -->
    <button class="boto-accions" onclick="alert('Hola!')">Clica'm</button>

    <!-- 6. INPUT: L'etiqueta que no podem oblidar per rebre dades -->
    <input type="text" placeholder="Escriu el teu nom..." id="entrada-usuari">

</div>

<style>
    /* Fem servir la CLASSE (.) per estils compartits */

    .boto-accions { background: #e74c3c; color: white; border: none; padding: 10px; }

VI

L'Art d'Endreçar l'Espai (Flexbox)

display:flex — el contenidor elàstic

Com fem que el menú estigui a l'esquerra i el contingut a la dreta? Amb display: flex, el contenidor es torna "elàstic" i ordena els seus elements de manera automàtica.

bloc-VI-flexbox.html 
<!-- El contenidor PARE defineix les regles -->
<div class="zona-flex">
    <div class="targeta">🍎 Recurs A</div>
    <div class="targeta">📚 Recurs B</div>
    <div class="targeta">💻 Recurs C</div>
</div>

<style>
    .zona-flex {
        display: flex;                  /* Activem el mode flexible */
        justify-content: space-around;  /* Reparteix l'espai entre elements */
        align-items: center;            /* Centra verticalment */
        flex-wrap: wrap;                /* Si no caben, salten de línia (Mòbil!) */
        gap: 20px;                      /* Distància de seguretat entre caixes */
        background: #f8f9fa;
        padding: 30px;
        border: 1px dashed #ccc;
    }

    .targeta {
        background: white;
        border-bottom: 4px solid #0d6efd;
        padding: 20px;
        flex: 1;              /* Totes intenten mesurar el mateix */
        min-width: 200px;     /* Però no es fan més petites que 200px */
        text-align: center;
        box-shadow: 0 4px 8px rgba(0,0,0,0.1);
    }
</style>

VII

El Cervell de la Web (Validació amb JavaScript)

EventListeners, DOM i lògica condicional

JavaScript ens permet comprovar les dades abans d'enviar-les. En aquest exemple, el botó d'enviar estarà bloquejat (desactivat) fins que l'usuari escrigui un correu electrònic amb el format correcte.

bloc-VII-validacio.html 
<!-- Formulari simple de Bootstrap -->
<div class="p-4 border rounded bg-light">
    <div class="mb-3">
        <label for="emailInput" class="form-label">Correu electrònic d'escola:</label>
        <input type="email" class="form-control" id="emailInput" placeholder="[email protected]">
        <div id="avis-error" class="text-danger mt-1" style="display:none;">L'email no és vàlid encara...</div>
    </div>
    
    <button id="botoEnviar" class="btn btn-primary" disabled>Enviar dades</button>
</div>

<script>
    const inputEmail = document.getElementById('emailInput');
    const boto = document.getElementById('botoEnviar');
    const avis = document.getElementById('avis-error');

    // Escoltador d'esdeveniments: "quan l'usuari escrigui..."
    inputEmail.addEventListener('input', function() {
        const valor = inputEmail.value;
        
        // Regla de validació simple (conté @ i .)
        const esValid = valor.includes('@') && valor.includes('.');

        if (esValid) {
            boto.disabled = false; // Activem el botó
            avis.style.display = 'none';
            inputEmail.classList.add('is-valid');
            inputEmail.classList.remove('is-invalid');
        } else {
            boto.disabled = true; // Bloquegem el botó
            avis.style.display = 'block';
            inputEmail.classList.add('is-invalid');
            inputEmail.classList.remove('is-valid');
        }
    });
</script>

VIII

Connexió Real (Notícies del Diari ARA via API)

Fetch, JSON asíncron, CORS i creació dinàmica de DOM

La web pot "llegir" altres mitjans. Aquí connectem amb l'RSS de l'Ara.cat, processem les dades i les maquetem amb imatges i descripcions en temps real. Hem de fer servir un proxy (rss2json) per evitar el bloqueig CORS del navegador.

bloc-VIII-api-noticies.html 
<div class="container mt-4">
    <h3 class="mb-4 text-danger border-bottom pb-2">Últimes notícies: Diari ARA</h3>
    <div id="noticiari" class="row g-4">
        <p class="text-muted">Connectant amb el diari...</p>
    </div>
</div>

<script>
async function carregarAra() {
    const contenidor = document.getElementById('noticiari');
    const rssUrl = 'https://www.ara.cat/rss/latest/';
    // Fem servir un convertidor RSS→JSON gratuït per evitar bloquejos (CORS)
    const apiProxy = `https://api.rss2json.com/v1/api.json?rss_url=${encodeURIComponent(rssUrl)}`;

    try {
        const resposta = await fetch(apiProxy);
        const dades = await resposta.json();

        contenidor.innerHTML = ""; // Netegem el missatge de càrrega

        // Agafem les primeres 3 notícies
        dades.items.slice(0, 3).forEach(item => {
            const col = document.createElement('div');
            col.className = "col-md-4";
            const imatge = item.enclosure?.link || item.thumbnail || 'https://picsum.photos/300/200';

            col.innerHTML = `
                <div class="card h-100 shadow-sm">
                    <img src="${imatge}" class="card-img-top" style="height:180px;object-fit:cover;" alt="Foto">
                    <div class="card-body">
                        <h5 class="card-title">${item.title}</h5>
                        <p class="card-text text-muted small">${item.description.substring(0,100)}...</p>
                        <a href="${item.link}" target="_blank" class="btn btn-outline-danger btn-sm">Llegir més</a>
                    </div>
                </div>`;
            contenidor.appendChild(col);
        });
    } catch(err) {
        contenidor.innerHTML = `<p class="text-danger">Error al connectar: ${err.message}</p>`;
    }
}

carregarAra(); // Executem la funció en carregar la pàgina
</script>

Secció 03

Glossari Vibe Coding: Conceptes per Construir una Web

🧠

Aquests són els conceptes que has de conèixer per demanar-li a una IA exactament el que vols. Com més vocabulari tècnic facis servir al teu prompt, millor serà el resultat. Passa el ratolí per sobre de cada targeta per veure un exemple de codi o ús.

— conceptes
<!DOCTYPE html>
Estructura
Declara al navegador que el document és HTML5. Ha d'anar sempre a la primera línia del fitxer, abans de tot.
<!DOCTYPE html>
<html lang="ca">...
<head>
Estructura
Conté tot el que el navegador necessita però l'usuari no veu: títol, fonts, CSS, metadades i scripts de configuració.
<head>
<title>La meva web</title>
</head>
charset="UTF-8"
Estructura
Defineix la codificació de caràcters. Imprescindible perquè els accents, la ç i els emojis es mostrin correctament a tots els sistemes.
<meta charset="UTF-8">
viewport
Estructura
Instrucció que indica als dispositius mòbils que adaptin l'amplada de la pàgina a la mida real de la pantalla. Sense això, el mòbil mostra la web en miniatura.
<meta name="viewport"
content="width=device-width,
initial-scale=1.0">
HTML Semàntic
Estructura
Etiquetes HTML5 amb significat: <header>, <main>, <section>, <article>, <aside>, <footer>, <nav>. Ajuden als cercadors i als lectors de pantalla.
<header>...</header>
<main>
<section>...</section>
</main>
<footer>...</footer>
lang="ca"
Estructura
Indica l'idioma principal de la pàgina. Millora el SEO i permet als lectors de pantalla pronunciar correctament el text.
<html lang="ca"> /* català */
<html lang="es"> /* castellà */
<html lang="en"> /* anglès */
Flexbox
CSS
Sistema de maquetació unidimensional. El contenidor pare amb display:flex controla com es col·loquen els fills: en fila, en columna, centrats, separats.
display: flex;
justify-content: space-between;
align-items: center;
gap: 1rem;
CSS Grid
CSS
Sistema de maquetació bidimensional per crear graelles complexes amb files i columnes. Ideal per a galeries, dashboards i layouts de pàgines completes.
display: grid;
grid-template-columns:
repeat(3, 1fr);
gap: 1.5rem;
Variables CSS
CSS
Variables reutilitzables que es defineixen a :root i s'usen a tot el document. Permeten canviar el color principal o la font en un sol lloc.
:root { --color-principal: #3b82f6; }
h1 { color: var(--color-principal); }
Box Model
CSS
Cada element HTML és una caixa amb: contingut + padding (espai interior) + border (vora) + margin (espai exterior). Entendre-ho resol el 80% dels problemes d'espaiat.
div {
padding: 1rem; /* interior */
border: 2px solid;
margin: 2rem; /* exterior */
}
Pseudo-classes
CSS
Selectores que apliquen estils en estats especials: :hover (ratolí a sobre), :focus (element actiu), :nth-child() (element N d'una llista).
a:hover { color: red; }
input:focus { border-color: blue; }
li:nth-child(2n) { background: #f5f5f5; }
Transitions
CSS
Fa que els canvis d'estil (color, mida, posició) es produeixin de manera suau en lloc d'instantàniament. La clau per a interaccions professionals.
button {
transition: background 0.3s ease,
transform 0.2s;
}
button:hover { transform: scale(1.05); }
@keyframes
CSS
Crea animacions CSS pures. Es defineix el punt inicial (from/0%) i final (to/100%) i el navegador calcula els passos intermedis automàticament.
@keyframes apareixer {
from { opacity: 0; transform: translateY(20px); }
to { opacity: 1; transform: translateY(0); }
}
.element { animation: apareixer .5s ease; }
position
CSS
Controla com es col·loca un element: fixed (queda fix en fer scroll), sticky (queda fix en arribar al límit), absolute (relatiu al pare).
header { position: sticky; top: 0; }
.overlay { position: fixed; inset: 0; }
.badge { position: absolute; top: -5px; }
Google Fonts
Tipografia
Biblioteca gratuïta de Google amb centenars de tipografies professionals. Es carreguen via CDN amb un link al <head> sense instal·lar res.
<link href="https://fonts.googleapis.com/
css2?family=Inter:wght@400;700
&display=swap" rel="stylesheet">
body { font-family: 'Inter', sans-serif; }
font-size + clamp()
Tipografia
clamp(mínim, preferit, màxim) crea textos que s'adapten a la pantalla de manera fluida sense media queries. La solució moderna per a títols responsius.
h1 {
font-size: clamp(1.5rem, 5vw, 3rem);
/* Mínim 1.5rem, ideal 5vw, màxim 3rem */
}
font-weight
Tipografia
Gruix de la tipografia. Va de 100 (ultra prim) a 900 (ultra negreta). Els valors més usats: 400 (normal), 600 (semibold), 700 (bold).
p { font-weight: 400; } /* normal */
h2 { font-weight: 700; } /* bold */
.hero { font-weight: 900; } /* black */
line-height
Tipografia
Espai entre línies de text. Un valor de 1.6–1.8 millora molt la llegibilitat en paràgrafs llargs. Valors sense unitat (com 1.6) s'escalen proporcionalment.
body { line-height: 1.7; } /* còmode */
h1 { line-height: 1.1; } /* titulars ajustats */
code { line-height: 1.5; }
Bootstrap 5
Llibreries
El framework CSS més popular. Inclou un sistema de grid de 12 columnes, components (cards, modals, navbar) i classes utilitàries. Es carrega via CDN.
<link href="https://cdn.jsdelivr.net/npm/
bootstrap@5.3.0/dist/css/bootstrap.min.css"
rel="stylesheet">
<div class="container row"></div>
Tailwind CSS
Llibreries
Framework CSS "atomic": en lloc de components, ofereix milers de classes utilitàries petites que es combinen directament al HTML. Molt popular en vibe coding.
<button class="bg-blue-500
text-white px-4 py-2
rounded-lg hover:bg-blue-700">
Enviar
</button>
Chart.js
Llibreries
Llibreria JavaScript per crear gràfiques interactives (barres, línies, sectors, radar) amb molt poc codi. Perfecte per a dashboards de dades.
<canvas id="grafica"></canvas>
<script>
new Chart(document.getElementById('grafica'), {
type: 'bar', data: {...}
});
</script>
Swiper.js
Llibreries
La millor llibreria per crear sliders, carrusels i galeries tàctils. Funciona perfectament en mòbils i és molt fàcil de configurar via CDN.

Font Awesome
Llibreries
Biblioteca d'icones vectorials (SVG) que es poden usar com a text CSS. Inclou milers d'icones de UI, xarxes socials, tecnologia i molt més.
<i class="fa-solid fa-house"></i>
<i class="fa-brands fa-github"></i>
<i class="fa-regular fa-envelope"></i>
CDN
Llibreries
Content Delivery Network: xarxa de servidors distribuïts que serveix fitxers (CSS, JS, fonts) ràpidament des del servidor més proper a l'usuari. Evita descarregar fitxers.
<!-- Sense CDN: has de descarregar -->
<link rel="stylesheet" href="bootstrap.css">
<!-- Amb CDN: carrega directe d'internet -->
<link href="https://cdn.jsdelivr.net/...">
DOM
JavaScript
Document Object Model: el "mapa" de tots els elements HTML que el JavaScript pot llegir i modificar. querySelector és la porta d'entrada.
const titol = document.querySelector('h1');
titol.textContent = 'Nou text!';
titol.style.color = 'red';
titol.classList.add('actiu');
EventListeners
JavaScript
Funcions que "escolten" accions de l'usuari: clic, tecla premuda, scroll, enviament de formulari. La base de tota la interactivitat web.
document.querySelector('#boto')
.addEventListener('click', function() {
alert('Has fet clic!');
});
const / let
JavaScript
const declara una variable que no canviarà (immutable). let declara una variable que sí pot canviar. Usa sempre const per defecte.
const nom = 'Reus'; // no canvia
let comptador = 0; // pot canviar
comptador = comptador + 1;
Arrow Functions
JavaScript
Sintaxi moderna i concisa per declarar funcions. S'usen molt en callbacks, forEach, map i addEventListener. La sintaxi és () => {}.
// Clàssica
function saludar(nom) { return 'Hola ' + nom; }
// Arrow (equivalent)
const saludar = (nom) => 'Hola ' + nom;
if / else
JavaScript
La lògica de presa de decisions del codi. Permet executar codi diferent depenent d'una condició (cert/fals). És la base de tota la validació.
if (edat >= 18) {
console.log('Major d\'edat');
} else {
console.log('Menor d\'edat');
}
Arrays & forEach
JavaScript
Un array és una llista de valors. forEach permet recórrer cada element per fer alguna acció. Essencial per mostrar llistes de dades dinàmicament.
const colors = ['vermell', 'blau', 'verd'];
colors.forEach(color => {
console.log(color);
});
localStorage
JavaScript
Permet guardar dades al navegador de l'usuari de manera persistent (sobreviu a tancar el navegador). Ideal per a preferències, temes, carret de compra senzill.
localStorage.setItem('tema', 'fosc');
const tema = localStorage.getItem('tema');
localStorage.removeItem('tema');
Fetch API
API & Dades
Funció nativa de JavaScript per fer peticions HTTP a servidors externs. Retorna una Promesa. La forma moderna de connectar-se a APIs i serveis web.
const resposta = await fetch('https://api.exemple.com/dades');
const dades = await resposta.json();
console.log(dades);
async / await
API & Dades
Sintaxi que permet escriure codi asíncron (que espera una resposta externa) de manera llegible, com si fos síncron. async marca la funció, await espera el resultat.
async function carregarDades() {
const res = await fetch('/api/noticies');
const json = await res.json();
return json;
}
JSON
API & Dades
JavaScript Object Notation: el format universal d'intercanvi de dades a Internet. Les APIs retornen JSON, i JavaScript el llegeix de manera nativa.
/* Format JSON */
{
"nom": "Reus",
"poblacio": 106000,
"comarques": ["Baix Camp"]
}
CORS
API & Dades
Cross-Origin Resource Sharing: mecanisme de seguretat del navegador que bloqueja les peticions a dominis externs. Sol solucionar-se amb un proxy (rss2json, allorigins).
/* Error típic CORS */
Access-Control-Allow-Origin blocat

/* Solució: usar un proxy */
fetch('https://api.allorigins.win/raw?url=' + encodeURIComponent(url))
RSS Feed
API & Dades
Format XML per subscriure's a actualitzacions de webs (notícies, blogs, podcasts). Molts diaris i serveis ofereixen una URL RSS que podem llegir des del nostre codi.
/* URL RSS d'exemple */
https://www.ara.cat/rss/latest/

/* Convertir a JSON (evita CORS) */
https://api.rss2json.com/v1/api.json?rss_url=...
APIs Públiques
API & Dades
Serveis externs que ofereixen dades gratuïtament: OpenWeatherMap (temps), Unsplash (fotos), JSONPlaceholder (proves), PokéAPI (dades). Requereixen una API Key.
/* Temps de Reus */
fetch('https://api.openweathermap.org/data/2.5/weather?q=Reus&appid=LA_TEVA_KEY')

<img> + srcset
Multimèdia
srcset permet servir imatges de diferent resolució depenent de la pantalla. loading="lazy" retarda la càrrega fins que l'imatge és visible.
<img
src="foto.jpg"
srcset="foto-2x.jpg 2x"
alt="Descripció"
loading="lazy">
object-fit
Multimèdia
Controla com s'adapta una imatge (o vídeo) dins del seu contenidor. cover omple el contenidor mantenint proporcions (pot retallar). contain mostra tot.
img {
width: 100%;
height: 200px;
object-fit: cover; /* ← retalla però omple */
object-position: center;
}
<video> / <audio>
Multimèdia
Etiquetes HTML5 natives per incrustar vídeo i àudio sense plugins. autoplay muted és el truc per a vídeos de fons (el navegador requereix el silenci).
<video autoplay muted loop playsinline>
<source src="video.mp4" type="video/mp4">
</video>
<input type>
Formularis
L'etiqueta més versàtil de HTML. Amb l'atribut type canvia de comportament: text, email, password, number, date, color, range, file, checkbox.
<input type="email" required>
<input type="range" min="0" max="100">
<input type="color" value="#3b82f6">
<input type="date">
RegEx (Validació)
Formularis
Expressions regulars per comprovar si un text segueix un patró (email, telèfon, DNI). Complexes però molt potents per a validació de formularis.
/* Validar email */
const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
const esEmail = regex.test('nom@escola.cat');
// → true
label + placeholder
Formularis
<label for> vincula un text descriptiu a un camp. placeholder mostra un text d'exemple dins del camp (desapareix en escriure). El label millora l'accessibilitat.
<label for="correu">Correu:</label>
<input id="correu" type="email"
placeholder="nom@escola.cat"
required>
Metadades Open Graph
SEO & Accés
Etiquetes meta que controlen com apareix la teva web quan es comparteix a WhatsApp, Twitter o LinkedIn: títol, descripció i imatge de previsualització.
<meta property="og:title" content="La meva web">
<meta property="og:description" content="Aprèn HTML">
<meta property="og:image" content="preview.jpg">
alt (text alternatiu)
SEO & Accés
Text descriptiu de les imatges per a persones amb discapacitat visual i per als cercadors. Si la imatge no carrega, es mostra el text alt. Obligatori per bona pràctica.
<!-- ❌ Incorrecte -->
<img src="foto.jpg">
<!-- ✅ Correcte -->
<img src="foto.jpg" alt="Alumnes treballant en grup">
ARIA roles
SEO & Accés
Accessible Rich Internet Applications: atributs per descriure elements dinàmics als lectors de pantalla. aria-label és el més usat per a botons sense text visible.
<button aria-label="Tancar menú">✕</button>
<nav role="navigation"></nav>
<div aria-live="polite">Missatge nou</div>
Media Queries
SEO & Accés
Regles CSS que apliquen estils depenent de la mida de pantalla. Permeten crear dissenys responsius que s'adapten a mòbil, tauleta i escriptori.
/* Per a pantalles de menys de 768px (mòbil) */
@media (max-width: 768px) {
.columnes { flex-direction: column; }
.menu { display: none; }
}
Favicon
SEO & Accés
La petita icona que apareix a la pestanya del navegador i als marcadors. Es declara al <head> amb un link. El format modern és SVG o PNG de 32×32px.
<link rel="icon" type="image/png"
href="/favicon.png">
<!-- O amb emoji (truc ràpid) -->
<link rel="icon" href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg'><text y='32' font-size='32'>📘</text></svg>">

Eines
Les eines de desenvolupador del navegador. Permeten inspeccionar l'HTML, modificar el CSS en viu, veure errors de JavaScript i analitzar les peticions de xarxa. Imprescindible.
