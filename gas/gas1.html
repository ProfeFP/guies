Quan programem amb Google Apps Script, és tentador llançar-se directament a escriure codi. Però igual que un arquitecte no construeix sense plànols, no hauríem de codificar sense haver dissenyat primer on i com es guardaran les dades. Aquesta guia és el punt de partida de tot el sistema.

🧠 Analogia: Imagina un gran arxivador físic per a les Jornades. Necessites carpetes separades: una per als inscrits, una per al programa, una per a les reserves de sala... A Google Sheets cada carpeta és una pestanya (fulla) del Spreadsheet.
BLOC I

Bloc I — Arquitectura General del Sistema

El nostre sistema gira al voltant d'un sol Google Spreadsheet que actua com a base de dades central. Google Forms l'alimenta automàticament i GAS llegirà i escriurà sobre ell per automatitzar totes les tasques.

1.2 Les 5 fulles del Spreadsheet

👥 INSCRITS

Registre de participants. S'omple des del Formulari automàticament.

📅 ACTIVITATS

Catàleg d'activitats per franja. GAS hi escriu els aforaments.

🚪 SALA_ESPECIAL

Agenda de 30 min. Generada per GAS automàticament.

📈 RESUM

Estadístiques globals calculades per GAS.

⚙️ CONFIG

Paràmetres generals: nom, data, lloc, emails.

1.3 Crea el Spreadsheet

1
Ves a drive.google.com → + Nou → Full de càlcul de Google.
2
Posa-li el nom: "Gestió Jornades Docents 2025".
3
Reanomena la primera pestanya INSCRITS (clic dret → Canvia el nom).
4
Crea les 4 pestanyes restants: ACTIVITATS, SALA_ESPECIAL, RESUM, CONFIG.
5
Opcional: assigna un color diferent a cada pestanya per identificar-les ràpidament.
💡 Conveni de noms: Usa sempre MAJÚSCULES sense espais per a les pestanyes. GAS distingeix entre majúscules i minúscules: getSheetByName("INSCRITS") no trobarà "Inscrits".
Exercicis del Bloc I

BLOC II

Bloc II — El Formulari d'Inscripcions

El Google Form és el punt d'entrada dels participants. Cada enviament crea automàticament una nova fila a INSCRITS, eliminant la introducció manual de dades i els errors de transcripció.

2.1 Crea el formulari des del Spreadsheet

1
Des del Spreadsheet: menú Insereix → Formulari. Crea un Forms directament vinculat.
2
Títol: "Inscripció Jornades Docents 2025". Afegeix data i lloc a la descripció.
3
Afegeix els 9 camps de la taula de sota. Activa Obligatori als essencials.
4
A Respostes del formulari comprova que apareix la icona verda del Sheets (vincle actiu).

2.2 Camps del formulari

# Camp Tipus Oblig. Notes
1 Nom i Cognoms Resposta curta ✅ Nom complet
2 Centre Educatiu Resposta curta ✅ Institut o centre
3 Correu electrònic Resposta curta ✅ Activa «Validació → Email»
4 Telèfon de contacte Resposta curta ❌ Per a emergències el dia de les jornades
5 Activitat Franja M1 (9:30–11:30) Opció múltiple ✅ Opcions: A, B, C, D
6 Activitat Franja M2 (12:00–14:00) Opció múltiple ✅ Opcions: A, B, C, D
7 Activitat Franja T (16:00–18:00) Opció múltiple ✅ Opcions: A, B, C, D
8 Vols reservar la Sala Especial? Opció múltiple ✅ Sí / No
9 Comentaris o necessitats especials Resposta llarga ❌ Accessibilitat, dietes...
⚠️ Les opcions A, B, C, D han de coincidir exactament amb els valors de la fulla ACTIVITATS. Recomanació: posa el codi i el títol: "A – Robòtica educativa". GAS fa la comparació caràcter per caràcter.
Exercicis del Bloc II
1 Crea el formulari complet amb els 9 camps. Envia'l 3 vegades amb dades fictícies i comprova que apareixen files a INSCRITS.
2 Activa la validació d'email al camp 3 i intenta enviar amb un email invàlid. Comprova que el formulari no ho permet.
3 Afegeix una pàgina de confirmació final: "Gràcies! Rebràs un correu de confirmació en les properes hores."

BLOC III

Bloc III — Fulla «INSCRITS»

La fulla INSCRITS és el registre central de participants. Les columnes A–J les omple el formulari. La columna K la gestiona GAS per evitar enviar el correu de confirmació dues vegades.

3.1 Estructura de columnes

Col. Nom del camp Tipus Origen Descripció
A Marca_Temporal DateTime FORMS Data i hora de l'enviament (automàtic)
B Nom_Cognoms String FORMS Nom complet del participant
C Centre_Educatiu String FORMS Institut o centre d'on prové
D Email String FORMS Correu electrònic — clau per a notificacions
E Telefon String FORMS Telèfon (opcional)
F Activitat_M1 String FORMS Opció triada 9:30–11:30 (A/B/C/D)
G Activitat_M2 String FORMS Opció triada 12:00–14:00 (A/B/C/D)
H Activitat_T String FORMS Opció triada 16:00–18:00 (A/B/C/D)
I Vol_Sala_Especial String FORMS «Sí» o «No»
J Comentaris String FORMS Observacions lliures (pot estar buit)
K Confirmacio_Enviada Boolean GAS TRUE quan GAS ha enviat el correu
⚠️ No modifiquis les columnes A–J manualment. Qualsevol canvi pot trencar la sincronització amb el Formulari. Afegeix camps nous a partir de la columna L.

BLOC V

Bloc V — Fulla «SALA_ESPECIAL»

La Sala Especial es pot reservar en intervals de 30 minuts de les 9:00 a les 18:30 (19 intervals). La fulla és una agenda que GAS generarà automàticament, amb totes les franges i el seu estat.

5.1 Estructura de columnes

Col. Nom Tipus Origen Descripció
A Hora_Inici String GAS Inici de l'interval (ex: «09:00»)
B Hora_Fi String GAS Fi de l'interval (ex: «09:30»)
C Responsable String Manual Nom de la persona que reserva
D Grup_Activitat String Manual Descripció de l'ús de la sala
E Email_Contacte String Manual Correu del responsable
F Estat String Manual/GAS DISPONIBLE o RESERVAT

5.2 Exemple visual

Hora_Inici Hora_Fi Responsable Grup / Activitat Email Estat
09:00 09:30 DISPONIBLE
09:30 10:00 M. Gómez Coordinació TIC [email protected] RESERVAT
10:00 10:30 M. Gómez Coordinació TIC [email protected] RESERVAT
10:30 11:00 DISPONIBLE
... (fins les 18:30) ... DISPONIBLE

5.3 Avanç: GAS generarà les 19 files automàticament

JavaScript — generarAgendaSala() · Avanç Guia 2
function generarAgendaSala() {
  const fulla = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName("SALA_ESPECIAL");
  if (fulla.getLastRow() > 1)
    fulla.getRange(2,1,fulla.getLastRow()-1,6).clearContent();

  let min=9*60, fi=18*60+30, fila=2;
  while(min < fi) {
    fulla.getRange(fila,1).setValue(minAHora(min));
    fulla.getRange(fila,2).setValue(minAHora(min+30));
    fulla.getRange(fila,6).setValue("DISPONIBLE");
    min+=30; fila++;
  }
  Logger.log("Agenda generada: "+(fila-2)+" intervals");
}
function minAHora(m) {
  return `${String(Math.floor(m/60)).padStart(2,'0')}:${String(m%60).padStart(2,'0')}`;
}

BLOC VI

Bloc VI — Fulla «CONFIG»

La fulla CONFIG centralitza tots els paràmetres generals. GAS els llegirà d'aquí, mai del codi directament. Canviar la data o l'email de les properes jornades serà tan senzill com actualitzar un valor en aquesta fulla.

🧠 Analogia: CONFIG és el tauler de control de l'edifici. No cal trucar un electricista per canviar la temperatura: n'hi ha prou de girar un botó al tauler.
Paràmetre (col. A) Valor (col. B) Descripció (col. C)
NOM_JORNADES Jornades Docents 2025 Nom oficial de l'event
DATA_JORNADES 15/11/2025 Data de celebració
LLOC Institut XYZ · Sala d'Actes Ubicació
EMAIL_ORGANITZACIO [email protected] Email oficial de les Jornades
EMAIL_NOTIFICACIONS [email protected] Rep els informes automàtics
AFORAMENT_DEFAULT 25 Places per defecte per activitat
HORA_INICI_SALA 09:00 Primera franja de la Sala
HORA_FI_SALA 18:30 Última franja de la Sala
INTERVAL_SALA_MIN 30 Durada de cada franja (minuts)
ACTIU TRUE FALSE suspèn totes les automatitzacions
JavaScript — llegirConfig() · Avanç Guia 2
function llegirConfig() {
  const dades = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName("CONFIG").getDataRange().getValues();
  const cfg = {};
  for (let i=1; i<dades.length; i++)
    if (dades[i][0]) cfg[dades[i][0]] = dades[i][1];
  return cfg;
}

BLOC VIII

Bloc VIII — Fulla «RESUM»

La fulla RESUM és la vista de control de l'organitzador. No la omplim a mà: GAS la generarà cada cop que s'executi el script de resum. Aquí la preparem estructuralment.

8.1 Estructura prevista

Zona Contingut Qui ho omple
A1:B1 Títol del resum i data de l'última actualització GAS
A3:C3 Capçalera: Total inscrits / Places totals / % ocupació global Manual (títols)
A4:C4 Xifres globals calculades GAS
A6:E18 Taula d'activitats: ID, Titol, Inscrits, Places lliures, Estat GAS
A20:B25 Top 5 centres educatius amb més participants GAS
Ara ja estàs a punt per a la Guia 2

Amb la base de dades completa, a la Guia 2 obrirem l'IDE de Google Apps Script i escriurem les primeres funcions reals: un onOpen() per afegir un menú personalitzat al Sheets, funcions CRUD per llegir i escriure dades, i un doGet() per exposar les activitats com a API pública.

Exercicis del Bloc VIII
1 Crea la fulla RESUM amb els títols manuals de les zones A3:C3. Deixa-la buida, GAS l'omplirà a la Guia 2.
2 [Repte] Anota 3 preguntes que voldries que el RESUM respongués d'un cop d'ull (ex: «Quantes places queden a M1-A?»). Les programaràs a la Guia 2.
