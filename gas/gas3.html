BLOC I

Bloc I — Variables i Constants

Les variables i les constants són els contenidors d'informació del nostre programa. Cada vegada que el script necessita recordar alguna cosa —un nom, un número, un estat— ho guarda en una variable o constant.

1.1 let — valors que canvien

🧠 Analogia docent: let és com el comptador de places disponibles d'una activitat: comença amb un valor i va decreixent a mesura que s'apunten participants. Canvia constantment.
JavaScript — let: valors variables
let placesDisponibles = 25;
let nomParticipant    = "Anna Ferrer Bosch";
let confirmacioEnviada = false;

// El valor pot canviar durant l'execució
placesDisponibles = placesDisponibles - 1;  // S'ha inscrit algú
confirmacioEnviada = true;               // Ja hem enviat el correu

Logger.log("Places restants: " + placesDisponibles);  // 24
Logger.log("Confirmació enviada: " + confirmacioEnviada); // true

1.2 const — valors que no canvien

🧠 Analogia docent: const és com el nom de les Jornades o la data de celebració: es defineixen una vegada i no s'han de tornar a tocar. Per convenció, els noms de constants s'escriuen en MAJÚSCULES.
JavaScript — const: valors fixos
const NOM_JORNADES    = "Jornades Docents 2025";
const DATA_JORNADES   = "15/11/2025";
const AFORAMENT_MAX   = 25;
const EMAIL_ORG       = "[email protected]";

Logger.log("Benvinguts a " + NOM_JORNADES);

// AFORAMENT_MAX = 30; → ❌ Error! No es pot reasignar una const

// Constants per als índexs de columna (bon ús habitual en GAS)
const COL_NOM   = 1;   // Columna B d'INSCRITS
const COL_EMAIL = 3;   // Columna D d'INSCRITS

1.3 Àmbit (scope): on viu una variable

JavaScript — àmbit global vs local
// Variables GLOBALS: definides fora de qualsevol funció.
// Disponibles per a tots els scripts del projecte.
const FULL_INSCRITS = "INSCRITS";   // ← global: totes les funcions la veuen

function comptarInscrits() {
  // Variable LOCAL: només existeix dins d'aquesta funció
  let total = 0;

  const fulla = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName(FULL_INSCRITS); // ← usa la global
  total = fulla.getLastRow() - 1; // -1 per saltar la capçalera

  Logger.log("Total inscrits: " + total);
  return total;
}
// Logger.log(total); → ❌ Error! 'total' no existeix fora de la funció

Bloc II — Tipus de Dades

Cada valor que guardem en una variable té un tipus. JavaScript els gestiona automàticament, però conèixer-los evita molts errors quan llegim dades d'un Google Sheets.

2.2 Conversió de tipus: el problema del Sheets

⚠️ Quan GAS llegeix un Sheets, tots els valors arriben com a String! Hauràs de convertir-los si vols operar numèricament. Aquesta és una de les causes d'error més freqüents.
JavaScript — conversió de tipus (molt comú a GAS)
// Problema: la cel·la G2 d'ACTIVITATS conté "25" (string, no number!)
let aforamentCrua = "25";               // ve del Sheets
Logger.log(typeof aforamentCrua);       // "string" ← problema!
Logger.log(aforamentCrua + 5);         // "255" ← concatena, no suma!

// Solució: convertir a número
let aforament = Number(aforamentCrua);  // o parseInt(), parseFloat()
Logger.log(typeof aforament);           // "number" ✓
Logger.log(aforament + 5);             // 30 ✓

// Conversió de Boolean: "TRUE" del Sheets ≠ true de JavaScript
let confirmatCru = "TRUE";             // ve del Sheets com a string
let confirmat    = confirmatCru === "TRUE" || confirmatCru === true;
Logger.log(confirmat);                  // true ✓

// Template literals: la forma moderna de construir strings
let nom = "Laura", places = 7;
Logger.log(`Hola ${nom}, queden ${places} places!`);

BLOC III

Bloc III — Estructures de Control

Les estructures de control permeten al programa prendre decisions i executar codi diferent segons les condicions. Sense elles, tots els scripts farien sempre el mateix.

3.1 if / else if / else

Cas pràctic: classificar l'ocupació d'una activitat

Quan el script actualitza els aforaments, ha de decidir quins missatges enviar als organitzadors: avís si una activitat es queda sense places, informació si està a bon ritme, etc.

JavaScript — if/else: classificar ocupació
function classificarOcupacio(inscrits, aforamentMax) {
  const pct = (inscrits / aforamentMax) * 100;
  let estat, missatge;

  if (inscrits >= aforamentMax) {
    estat    = "Completa";
    missatge = "🔴 COMPLETA — llista d'espera disponible";
  } else if (pct >= 80) {
    estat    = "Quasi plena";
    missatge = "🟠 Quasi plena — queden poques places";
  } else if (pct >= 50) {
    estat    = "Oberta";
    missatge = "🟡 A bon ritme";
  } else {
    estat    = "Oberta";
    missatge = "🟢 Moltes places disponibles";
  }

  Logger.log(`${inscrits}/${aforamentMax} (${pct.toFixed(0)}%) → ${missatge}`);
  return estat;
}

// Proves
classificarOcupacio(25, 25);  // 🔴 COMPLETA
classificarOcupacio(21, 25);  // 🟠 Quasi plena
classificarOcupacio(14, 25);  // 🟡 A bon ritme
classificarOcupacio(5,  25);  // 🟢 Moltes places

3.2 Operadors lògics i de comparació

JavaScript — operadors: AND, OR, NOT, ternari
// ── Comparació ────────────────────────────────────────────────
let places = 5;
Logger.log(places === 5);   // true  (igual en valor I tipus)
Logger.log(places !== 10);  // true  (diferent)
Logger.log(places < 10);    // true  (menor que)
Logger.log(places >= 5);    // true  (major o igual)

// ── AND (&&): les dues condicions han de ser certes ───────────
let estaDisponible = places > 0 && !confirmacioEnviada;

// ── OR (||): n'hi ha prou amb una ────────────────────────────
let necessitaAvis = places === 0 || places === 1;

// ── NOT (!): inverteix ────────────────────────────────────────
let noPotEntrar = !estaDisponible;

// ── Operador ternari: if/else en una línia ────────────────────
const etiqueta = places > 0 ? "Oberta" : "Completa";
//               condició       si cert    si fals

// Molt útil per construir missatges ràpids:
const icon = places === 0 ? "🔴" : places < 5 ? "🟠" : "🟢";
Logger.log(icon + " " + etiqueta);

BLOC IV

Bloc IV — Bucles i Iteració

Els bucles permeten repetir una acció per a cada element d'una llista. Quan GAS llegeix les 20 files d'inscrits d'un Sheets i processa cadascuna, ho fa gràcies a un bucle.

🧠 Analogia docent: Un bucle és com un professor que corregeix exàmens un per un: agafa el primer, el corregeix, el deixa; agafa el segon, el corregeix, el deixa... fins que no en queda cap.

4.1 for clàssic — quan saps quantes iteracions

JavaScript — for: processar files d'inscrits
function comptarPerActivitat() {
  const fulla  = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("INSCRITS");
  const dades  = fulla.getDataRange().getValues();
  let comptador = { M1:{A:0,B:0,C:0,D:0}, M2:{A:0,B:0,C:0,D:0}, T:{A:0,B:0,C:0,D:0} };

  // Comença a i=1 per saltar la fila de capçalera
  for (let i = 1; i < dades.length; i++) {
    const fila = dades[i];
    if (!fila[1]) continue;  // salta files buides

    const opcM1 = fila[5];  // col F: activitat Matí 1
    const opcM2 = fila[6];  // col G: activitat Matí 2
    const opcT  = fila[7];  // col H: activitat Tarda

    if (comptador.M1[opcM1] !== undefined) comptador.M1[opcM1]++;
    if (comptador.M2[opcM2] !== undefined) comptador.M2[opcM2]++;
    if (comptador.T[opcT]   !== undefined) comptador.T[opcT]++;
  }

  Logger.log("Matí 1:  " + JSON.stringify(comptador.M1));
  Logger.log("Matí 2:  " + JSON.stringify(comptador.M2));
  Logger.log("Tarda:   " + JSON.stringify(comptador.T));
  return comptador;
}

4.3 while — quan no saps d'entrada quantes iteracions

JavaScript — while: generar franges de 30 min
function generarFranges() {
  let minuts = 9 * 60;        // Comença a les 9:00 (en minuts)
  const fi  = 18 * 60 + 30; // Acaba a les 18:30
  const franges = [];

  while (minuts < fi) {
    const h   = Math.floor(minuts / 60);
    const min = minuts % 60;
    const hLabel = `${String(h).padStart(2,'0')}:${String(min).padStart(2,'0')}`;
    franges.push(hLabel);
    minuts += 30;  // avança 30 minuts
  }

  Logger.log("Total de franges: " + franges.length);  // 19
  Logger.log("Primera: " + franges[0]);              // "09:00"
  Logger.log("Última:  " + franges[franges.length-1]); // "18:00"
  return franges;
}

BLOC V

Bloc V — Funcions

Les funcions són blocs de codi reutilitzables amb un nom propi. Un bon script de GAS és una col·lecció de funcions petites, cadascuna amb una responsabilitat clara. Si una funció fa massa coses, cal dividir-la.

🧠 Analogia docent: Les funcions són com les unitats didàctiques: cadascuna té un objectiu concret, uns materials d'entrada (paràmetres) i uns resultats de sortida (return). Si una unitat intenta cobrir-ho tot, els alumnes s'hi perden.

5.1 Anatomia d'una funció

JavaScript — parts d'una funció ben construïda
// ┌─ paraula clau
// │     ┌─ nom descriptiu (camelCase)
// │     │              ┌─ paràmetres d'entrada
// ↓     ↓              ↓                          
function calcularPlacesLliures(aforamentMax, inscrits) {
  // Cos: la lògica de la funció
  const places = aforamentMax - inscrits;
  if (places < 0) return 0;  // cas excepcional
  return places;               // ← valor de retorn
}

// Crida de la funció: passem els arguments
let lliures = calcularPlacesLliures(25, 18);
Logger.log("Places lliures: " + lliures);  // 7

// Valors per defecte als paràmetres
function construirIdActivitat(franja, opcio = "A") {
  return franja + "-" + opcio;   // "M1-A", "T-B"...
}
Logger.log(construirIdActivitat("M1"));      // "M1-A" (opcio = "A" per defecte)
Logger.log(construirIdActivitat("T", "C"));  // "T-C"

5.2 Funcions que retornen objectes

JavaScript — retornar un objecte d'estadístiques
function estadistiquesJornades() {
  const inscrits   = llegirInscrits();
  const activitats = llegirActivitats();

  const totalPlaces = activitats.reduce((s, a) => s + a.aforamentMax, 0);
  const volSala     = inscrits.filter(p => p.volSala === "Sí").length;

  return {
    totalInscrits:  inscrits.length,
    totalPlaces,
    pctOcupacio:    Math.round(inscrits.length / totalPlaces * 100),
    volSalaEspecial: volSala,
    actualitzat:    new Date().toLocaleString("ca-ES")
  };
}

// Ús: desestructurem el resultat

BLOC VI

Bloc VI — Arrays i els seus Mètodes

Un array és una llista ordenada de valors. Quan GAS llegeix les files d'un Sheets, obté un array d'arrays. Dominar els mètodes d'array és la clau per escriure scripts nets i potents.

🧠 Analogia docent: Un array és com la llista de participants en paper: cada inscrit ocupa una posició numerada (índex 0, 1, 2...). Els mètodes filter, map i reduce són com operacions que fas sobre aquesta llista: "subratlla els del centre Miralles", "copia només els noms a una nova llista", "compta quants hi ha en total".

6.1 Creació i accés

JavaScript — arrays: creació i accés bàsic
// Arrays simples
const opcions  = ["A", "B", "C", "D"];
const franges  = ["M1", "M2", "T"];
const places   = [50, 25, 30, 20];

Logger.log(opcions[0]);        // "A" (índex 0 = primer element)
Logger.log(opcions.length);    // 4
Logger.log(opcions.includes("C")); // true

// Afegir i treure elements
let inscritsAvui = ["Anna", "Jordi"];
inscritsAvui.push("Maria");    // afegeix al final → ["Anna","Jordi","Maria"]
inscritsAvui.unshift("Pere"); // afegeix al principi
inscritsAvui.pop();            // treu l'últim

// Array 2D: el que retorna getValues() del Sheets
const dades = [
  ["Nom",   "Email",          "Activitat"],  // fila 0 = capçalera
  ["Anna",  "[email protected]",   "A"],           // fila 1
  ["Jordi", "[email protected]",  "B"]            // fila 2
];
Logger.log(dades[1][0]);  // "Anna"  (fila 1, col 0)
Logger.log(dades[2][2]);  // "B"     (fila 2, col 2)

6.2 Els 5 mètodes essencials

JavaScript — map, filter, find, reduce, some
const inscrits = llegirInscrits();  // array d'objectes participant

// ── map: transforma cada element → nou array ──────────────────
const emails = inscrits.map(p => p.email);
// ["[email protected]", "[email protected]", ...]

const noms = inscrits.map(p => p.nom.split(" ")[0]);
// ["Anna", "Jordi", "Maria", ...]  ← primer nom

// ── filter: filtra elements → subconjunt ──────────────────────
const pendents   = inscrits.filter(p => !p.confirmat);
const volSala    = inscrits.filter(p => p.volSala === "Sí");
const deM1A      = inscrits.filter(p => p.activM1 === "A");
Logger.log("Inscrits a M1-A: " + deM1A.length);

// ── find: retorna el PRIMER que compleix la condició ──────────
const anna = inscrits.find(p => p.nom.includes("Anna"));
Logger.log(anna?.centre);  // "INS Can Puig - Badalona"

// ── reduce: acumula un resultat ───────────────────────────────
const activitats = llegirActivitats();
const totalPlaces = activitats.reduce((suma, a) => suma + a.aforamentMax, 0);
Logger.log("Total places: " + totalPlaces);  // 130 (suma tots els aforaments)

// ── some: retorna true si ALGUN element compleix ──────────────
const hiHaComplet = activitats.some(a => a.inscrits >= a.aforamentMax);
Logger.log("Alguna activitat completa? " + hiHaComplet);

BLOC VII

Bloc VII — Objectes i JSON

Un objecte agrupa dades relacionades sota un mateix nom. En el nostre sistema, cada participant, cada activitat i cada franja de la Sala Especial és un objecte. El format JSON és la manera estàndard d'intercanviar objectes entre sistemes (és el que retorna la nostra Web App).

🧠 Analogia docent: Un objecte és com la fitxa d'inscripció d'un participant: té camps amb nom (nom, email, activitats) i el seu valor corresponent. JSON és com enviar aquella fitxa per correu: es converteix a un format de text estàndard que qualsevol sistema pot llegir i reconstruir.

  Logger.log(`${i+1}. ${a.id} – ${Math.round(a.inscrits/a.aforamentMax*100)}%`)

BLOC VIII

Bloc VIII — Gestió d'Errors i Bones Pràctiques

Un bon script no és només el que funciona quan tot va bé: és el que gestiona els errors amb elegància i és fàcil de llegir i mantenir mesos després d'haver-lo escrit.

🧠 Analogia docent: El bloc try/catch és com una xarxa de seguretat en un trapezi: no s'espera caure, però si passa, atura la caiguda sense que tot l'espectacle s'aturi. Un script sense gestió d'errors és un script que para de treballar al primer obstacle; un amb try/catch registra el problema i continua processant la resta.

8.1 try / catch / finally

JavaScript — try/catch: gestió robusta d'errors
function enviarConfirmacioSegura(participant) {
  const cfg = llegirConfig();

  try {
    // Validem dades abans d'enviar
    if (!participant.email) throw new Error("Email buit per a: " + participant.nom);
    if (!participant.nom)   throw new Error("Participant sense nom a fila " + participant.fila);

    const assumpte = `✅ Confirmació – ${cfg["NOM_JORNADES"]}`;
    GmailApp.sendEmail(participant.email, assumpte, "...");
    Logger.log("✉️ Enviat a: " + participant.email);
    return true;

  } catch (error) {
    // Capturem l'error: el loguegem però NO parem tot el script
    Logger.log("⚠️ ERROR: " + error.message);
    GmailApp.sendEmail(cfg["EMAIL_NOTIFICACIONS"],
      "⚠️ Error al sistema Jornades", error.message);
    return false;

  } finally {
    // S'executa SEMPRE, hagi hagut error o no
    SpreadsheetApp.flush(); // força l'escriptura al Sheets
  }
}

function enviarConfirmacionsPendents() {
  const inscrits = llegirInscrits();
  const pendents = inscrits.filter(p => !p.confirmat);
  pendents.forEach(p => enviarConfirmacioSegura(p));
  Logger.log(`Enviats: ${pendents.length}`);
}
Les 7 regles del codi GAS professional
  1. Noms descriptius: enviarConfirmacionsPendents() en lloc de f1(). El nom ha d'explicar el que fa.
  2. Una funció, una responsabilitat: si una funció fa massa coses, divideix-la. Màxim 20-30 línies per funció.
  3. Constants per a tot el que és fix: mai escriguis dades[i][3] — usa dades[i][COL_EMAIL].
  4. try/catch a tot el que toca serveis externs: GmailApp, SpreadsheetApp, UrlFetchApp... sempre poden fallar.
  5. Comprova CONFIG[ACTIU] al principi de cada funció crítica: permet apagar el sistema sense tocar el codi.
  6. SpreadsheetApp.flush() per forçar escriptures: especialment en funcions llargues que escriuen moltes cel·les.
  7. Historial de versions: abans de fer canvis importants, crea una versió nova (Projecte → Historial de versions → Desa versió).

8.3 Logger.log vs console.log vs alert

JavaScript — quan usar cada mètode de sortida
// ── Logger.log(): per al/la DESENVOLUPADOR/A durant el codi ──
Logger.log("Debug: processant fila " + i);
Logger.log("Valor rebut: " + JSON.stringify(objecte));
// → Visible a: Vista → Registres d'execució de l'IDE

// ── console.log(): equivalent modern, visible a la consola ───

// ── SpreadsheetApp.getUi().alert(): per a l'USUARI FINAL ─────
SpreadsheetApp.getUi().alert("✅ Confirmacions enviades correctament!");

// ── toast(): missatge breu no intrusiu (recomanat) ────────────
SpreadsheetApp.getActiveSpreadsheet()
  .toast("Aforaments actualitzats", "✅ Jornades", 5);
// → Apareix com a notificació breu a la cantonada (5 segons)
