🧩 Personalització avançada

Skills · Instruccions Personalitzades
per a la IA a l'aula

Com donar memòria, context i personalitat a qualsevol model d'IA. Cada plataforma té el seu nom per a aquesta funció — Skills, GPTs, Gems, Instruccions de sistema — però totes fan el mateix: convertir una IA genèrica en un assistent a mida.

🤔 Programar sense escriure codi… és programar?

Durant dècades, programar significava dominar una sintaxi: saber on va el punt i coma, com funcionen els bucles, per què una variable de tipus string no es pot sumar directament a un enter. L'habilitat tècnica era la porta d'entrada — i moltes vegades, la barrera.

Avui, un docent que escriu un skill per a Claude, un GPT personalitzat per a Copilot o un Gem a Gemini està programant lògica, definint comportament, establint restriccions i dissenyant fluxos — tot en llenguatge natural. Sense codi. Però amb la mateixa intenció que sempre ha tingut un programador: fer que una màquina es comporti exactament com vols.

Aquesta nova realitat ens obliga com a docents a reformular preguntes fonamentals: Quines competències importa ara ensenyar? El pensament lògic, la capacitat de descompondre un problema, saber definir l'abast d'una tasca i comunicar-la amb precisió — tot allò que sempre ha estat el nucli de la programació — segueix sent imprescindible. El que canvia és l'eina. I el que no canvia és la necessitat de pensar bé abans de comunicar.

Els skills d'IA no simplifiquen la programació: la democratitzen. I obren la porta perquè més persones puguin construir, automatitzar i crear — amb o sense experiència tècnica prèvia. Per als docents, això no és una amenaça. És una oportunitat de recentrar l'ensenyament en allò que realment importa.

📖 Concepte

Què és un Skill (o instrucció personalitzada)?

Un skill és un fitxer de text — normalment en format Markdown — que conté instruccions detallades sobre com ha de comportar-se un model d'IA en un context concret. Defineix el rol, el to, les restriccions, el format de les respostes i fins i tot exemples de bones pràctiques. És, en essència, el codi font d'un assistent especialitzat.

Cada plataforma ha batejat aquesta funcionalitat d'una manera diferent, però el principi és idèntic: capes de context persistent que s'injecten automàticament a cada conversa sense que l'usuari hagi de repetir les instruccions cada vegada.

🌐 Plataformes

El nom canvia, la idea és la mateixa

Totes les grans plataformes d'IA ofereixen alguna forma de personalització persistent. Aquí tens les més rellevants en context educatiu:

🟣

Claude · Anthropic

🧩 Skills & System Prompts

Claude utilitza skills com a fitxers Markdown estructurats que es carreguen automàticament en un projecte. Permeten definir comportament, to, límits de contingut i fins i tot workflows multiestep. El sistema de projectes de Claude.ai integra els skills directament sense necessitat de codi.

Fitxer Markdown Per projectes Multiestep Claude.ai / API
🟢

ChatGPT · OpenAI

🤖 GPTs Personalitzats

Els GPTs permeten crear assistents amb nom, avatar, instruccions pròpies i fins i tot eines externes (Browsing, DALL·E, Code Interpreter). Es publiquen a la GPT Store i es poden compartir amb altres usuaris. La versió de Microsoft és Copilot Studio, que afegeix integració empresarial.

GPT Store Eines integrades Copilot Studio Compartible
💎

Gemini · Google

💎 Gems

Els Gems de Gemini Advanced permeten definir un assistent personalitzat amb nom, descripció i instruccions detallades. S'integren directament a l'espai de treball de Google (Docs, Drive, Gmail) i permeten accedir a informació dels documents personals com a context.

Gems Google Workspace Integració Drive Gemini Advanced
🔵

Microsoft Copilot

⚙️ Copilot Studio / Agents

Copilot Studio permet crear agents empresarials amb instruccions, connectors a dades corporatives i fluxos automatitzats. Per a docents en entorns Microsoft 365 és una eina poderosa: es pot crear un "Copilot de l'aula" amb accés als documents compartits del curs.

Microsoft 365 Agents Power Platform Empresarial
🔍

Perplexity

📋 Instruccions del perfil

Perplexity permet afegir instruccions personalitzades al perfil que s'apliquen a totes les cerques. Menys avançat que els altres sistemes, però útil per definir el nivell de detall, l'idioma preferit i l'estil de les respostes. Especialment interessant per la seva capacitat de cerca web en temps real.

Cerca web Instruccions de perfil Freemium
🖥️

LLMs locals (Ollama)

📝 Modelfile / System Prompt

En models locals com Ollama (Llama, Mistral, Phi…) la personalització es fa via Modelfile o passant un system prompt directament. Màxima privacitat i control: les instruccions queden al propi ordinador i cap dada surt a internet. Ideal per a entorns amb restriccions de privacitat de dades d'alumnes.

100% local Sense núvol Open source RGPD ✓

📥 Recurs

Skill de mostra: Rúbrica Docent

Com a exemple pràctic, aquí tens el skill rubrica-docent creat per a Claude. A partir de l'enunciat d'una activitat de programació, genera automàticament una rúbrica d'avaluació en quatre nivells (Excel·lent / Bé / Regular / Insuficient) adaptada als objectius de la tasca. Podeu descarregar els dos fitxers que composen el skill i afegir-los a qualsevol projecte de Claude.

🧩 rubrica-docent · Skill per a Claude

Dos fitxers: SKILL.md conté les instruccions que llegeix Claude sobre com funciona el sistema de skills. rubrica-docent.skill conté el comportament específic del generador de rúbriques: rol, format de sortida, nivells d'avaluació i criteris per a activitats de programació.

🍃 Nota pedagògica

Recomanació per a l'aula: proposeu als alumnes que creïn el seu propi skill personalitzat per a una tasca concreta del curs. El procés de definir instruccions clares, preveure casos límit i iterar fins que l'assistent es comporti exactament com volen és, en si mateix, un exercici de pensament computacional i comunicació precisa — amb o sense escriure codi. Comparar els resultats entre plataformes (Claude, ChatGPT, Gemini) afegeix una capa de pensament crític molt valuosa.
